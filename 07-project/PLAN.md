## Project Plan

### Class Criteria for Peer Review 
If I was taking the live version of the class then this would be the criteria I would use to review the project and have my own project reviewed on.

- Problem Description
- Cloud Usage
- Experiment Tracking and Model Registry
- Workflow Orchestration
- Model Deployment
- Monitoring
- Reproducibility
- Best Practices

### Problem Description

#### Answers these questions well:
- What is the problem you are trying to solve?
- What is the data you are using?
- What is the model you are using?
- What is the evaluation metric you are using?
- What is the training data you are using?
- What is the validation data you are using?
- What is the test data you are using?

### Cloud Usage
- Cloud provider and resources are used
- IaC tools are used to provision the resources (e.g. Terraform, CloudFormation, etc.)
- The project is developed and deployed on the cloud

### Experiment Tracking and Model Registry
- Experiment tracking is used
- Model registry is used
- The experiments are tracked in the experiment tracking system (e.g. MLflow, Weights & Biases, etc.)
- The models are registered in the model registry
- The experiments and models are tracked in the experiment tracking system
- BONUS: Data versioning is used

### Workflow Orchestration
- Workflow orchestration is used
- The workflows are orchestrated in the workflow orchestration system (e.g. Airflow, Prefect, etc.)
- BONUS: Try out a new workflow orchestration tool (Airflow, Argo, etc.)

### Model Deployment
- Model deployment is used
- The models are deployed in the model deployment system (e.g. Seldon, BentoML, etc.)
- The models are deployed in the model deployment system
- The models are deployed in the model deployment system
- BONUS: Try out a new model deployment tool (Seldon, BentoML, etc.)

### Monitoring
- Monitoring is used
- The monitoring is done in the monitoring system (e.g. Prometheus, Grafana, etc.)
- The monitoring is done in the monitoring system

### Reproducibility
- Reproducibility is used
- The project is reproducible
- BONUS: Try out a new reproducibility tool (ReproZip, ReproSchema, etc.)

### Best Practices
- Unit tests are used
- Integration tests are used
- Linter and/or code formatter is used
- Makefile is used
- Pre-commit hooks are used
- CI/CD pipeline is used
